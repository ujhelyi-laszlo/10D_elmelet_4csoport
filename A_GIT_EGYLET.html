GIT

A Git egy nyílt forráskódú, elosztott verziókövető rendszer, amelyet eredetileg Linus Torvalds hozott létre a Linux kernel fejlesztésének támogatására. A Git célja, hogy hatékonyan kezelje a nagy projekteket, és támogassa a nem lineáris fejlesztési folyamatokat.
A Git lehetővé teszi a fejlesztők számára, hogy dolgozzanak a saját lokális másolatukon, majd szinkronizálják a változtatásaikat egy központi tárolóval. Ez a decentralizált működési mód növeli a rugalmasságot és a hatékonyságot.

Git legfontosabb funkciói közé tartozik

  • A változások nyomon követése
  • A konfliktusok feloldása
  • A visszaállítás
  • A kód történetének megtekintése

A Git támogatja a branch-eket és a címkéket, amelyek segítségével a fejlesztők különböző verziókat és kiadásokat kezelhetnek.
A csoportmunkát támogató eszközök és online szolgáltatások, mint például a Github, a GitLab, és a Bitbucket, további funkcionalitást adnak a Githez. Ezek a platformok lehetővé teszik a kód megosztását, a projektmenedzsmentet, a hibakövetést és a dokumentációt. Ezek az eszközök elősegítik a csapatmunkát és a közösségi fejlesztést, lehetővé téve a fejlesztők számára, hogy együtt dolgozzanak, függetlenül attól, hogy hol találhatók a világban.

GITHUB

A GitHub platform fejlesztése 2007. október 19-én kezdődött. Az oldalt 2008 áprilisában indította el Tom Preston-Werner, Chris Wanstrath, P.J. Hyett és Scott Chacon miután pár hónappal előtte elérhetővé tették béta verzióként.
A GitHubon található projekteket a Git parancssorral lehet elérni és használni és minden alapvető Git parancs működik rajta. A GitHub továbbá lehetővé teszi regisztrált és nem regisztrált felhasználóknak is nyilvános tárolók böngészését az oldalon. Több asztali alkalmazást és Git plugint is létrehozott a GitHub és egyéb harmadik felek akik egyesülnek a platformmal.
Az oldal közösségi kommunikáció-szerű funkciókat is tartalmaz mint feed-ek, követők, wiki-k (a Gollum nevű wiki szoftvert használva) és egy közösségi hálózat grafikont, hogy kijelezze, hogyan dolgoznak a fejlesztők a saját verzióikon („forks”) egy tárolónak és melyik fork (és azon belül ág) a legújabb.
Ahhoz, hogy feltöltsünk az oldalra, regisztráció szükséges, de a nyilvános tárolókat bárki szabadon böngészheti és letöltheti. Egy regisztrált felhasználóval lehetőség nyílik megbeszélésekre, tárolók kezelésére, mások tárolóihoz való hozzájárulásra és a kódban történt változások megtekintésére. 2019. januárjától korlátlan privát ingyenes tárolót lehet létrehozni (3 hozzájárulóval projektenként). Korábban csak a nyilvános tárolók voltak ingyenesek.
2020. április 14-én a GitHub „minden fő GitHub funkciót” ingyenessé tett mindenkinek, beleértve a „privát tárolókat korlátlan hozzájárulókkal”.
Az alapvető szoftver, ami alátámasztja a GitHubot, az maga a Git, amelyet Linus Torvalds, a Linux készítője hozott létre. A további szoftvereket, amik a GitHub felhasználói felületét adják, Ruby on Rails és Erlang használatával készítették a GitHub Inc. fejlesztői, Wanstrath, Hyett és Preston-Werner.

A GitHubot nagyrészt kódolásra használják.

Forráskódtároláson kívül a GitHub a következő formátumokat és funkciókat nyújtja:

  • Dokumentáció, beleértve automatikusan létrehozott README fájlokat számos fajta Markdown-szerű fájlformátumban (lásd README fájlok GitHubon)
  • Kiadáskövetés (beleértve a szolgáltatáskéréseket) címkékkel, mérföldkövekkel, megbízottakkal és keresőmotorral
  • Wikik
  • Pull request a kódáttekintéshez és megjegyzésekhez
  • GitHub Actions, amely lehetővé teszi a folyamatos integrációt és a folyamatos telepítési folyamatokat a szoftver tesztelésére, kiadására és telepítésére harmadik fél webhelyeinek / platformjainak használata nélkül
  • Hozzájárulási előzmények
  • Grafikonok: impulzus, közreműködők, vállalások, kódfrekvencia, lyukasztókártya, hálózat, tagok
  • Integrációs könyvtár[10]
  • Egységes és osztott diff
  • E-mail-értesítések
  • Lehetőség arra, hogy feliratkozzon valakinek az értesítésekre, @ említve.[11]
  • Hangulatjelek[12]
  • GitHub oldalak : kis weboldalak tárolhatók a GitHub nyilvános adattáraiban. Az URL formátuma: https: // felhasználónév .github.io.[13]
  • Fájlokba beágyazott feladatlisták
  • A térinformatikai adatok megjelenítése
  • 3D render fájlok, amelyek előnézetben megtekinthetők egy új integrált STL fájlnézegető segítségével, amely a fájlokat "3D vászonon" jeleníti meg.[14] A nézetet a WebGL és a Three.js táplálja.
  • A Photoshop natív PSD formátuma megnézhető és összehasonlítható ugyanazon fájl korábbi verzióival.
  • PDF dokumentum néző
  • Biztonsági riasztások az ismert általános biztonsági résekről és expozíciókról a különböző csomagokban

GITLAB

GitLab egy nyílt forráskódú, végigvezető szoftverfejlesztési platform, amely beépített verziókövetéssel, problémakezeléssel, kódértékeléssel, CI/CD-vel és még sok mással rendelkezik1. A GitLab Inc., amely a GitLab-ot működteti, egy DevOps szoftvercsomagot fejleszt, biztosít és működtet2. A GitLab-ot saját szervereken, konténerben vagy felhőszolgáltatón lehet önjelölt módon futtatni1.
A GitLab Inc. 2014-ben alakult, hogy folytassa az ukrán fejlesztő, Dmytro Zaporozhets által 2011-ben indított nyílt forráskódú kódmegosztó platform fejlesztését2. A cég társalapítója, Sytse Sijbrandij kezdetben hozzájárult a projekthez, és úgy döntött, hogy üzletet épít köré2. A GitLab a platformját freemium modellben kínálja2.
A GitLab Inc. a távmunkát népszerűsítette alapítása óta, és a világ egyik legnagyobb teljes mértékben távoli munkát végző vállalataként ismert2. 2020-ra a cég 1300 embert foglalkoztatott 65 országban2.

A GitLab számos előnnyel rendelkezik más verziókövető rendszerekkel szemben. Íme néhány:

  1. Együttműködési eszközök: A GitLab beépített eszközökkel rendelkezik a csapatmunka támogatására, beleértve a problémakezelést és a kódértékelést1.
  2. CI/CD integráció: A GitLab beépített CI/CD (Continuous Integration/Continuous Deployment) eszközökkel rendelkezik, amelyek automatizálják a kódtesztelést és a kiadásokat1.
  3. Nyílt forráskódú: A GitLab nyílt forráskódú, ami azt jelenti, hogy a közösség hozzájárulhat a fejlesztéséhez1.
  4. Önjelölt telepítés: A GitLab-ot saját szervereken, konténerben vagy felhőszolgáltatón lehet önjelölt módon futtatni1.
  5. Hitelesítési szintek: A GitLab a szereplőknek megfelelően állíthat be és módosíthat engedélyeket különböző együttműködők számára2.

Ezek az előnyök a GitLab-ot vonzóvá teszik a fejlesztők számára, akik egy teljes körű, rugalmas és testreszabható verziókövető rendszert keresnek. Mindazonáltal, a legjobb választás mindig a projekt specifikus követelményeitől és preferenciáitól függ

BITBUCKET

A Bitbucket egy Git-alapú eszköz, amelyet kifejezetten csapatok számára terveztek, és kódhosztolást és együttműködést nyújt1. A Bitbucket piacvezető Jira- és Trello-integrációit úgy tervezték, hogy összehozza a teljes szoftvercsapatot a projekt végrehajtása érdekében1.
A csapatok egyetlen helyen képesek együtt dolgozni a kódoláson a koncepciótól a felhőig, automatizált teszteléssel minőségi kódokat létrehozni, valamint magabiztosan telepíteni a kódokat1. A Bitbucket lehetővé teszi a csapatok számára, hogy kezeljék a Jira ügyeket a beépített Jira felhasználói felületen, és amikor az ügykulcsot belefoglalják a commitokba, a Jira ügy állapotai automatikusan frissülnek2.
